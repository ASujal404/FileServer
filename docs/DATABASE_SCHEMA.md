# Database ER Diagram & Schema Specifications

## Entity Relationship (ER) Diagram

```mermaid
erDiagram
    USERS ||--o{ FILES : "owns"
    USERS ||--o{ AUDIT_LOGS : "performs"
    USERS ||--o{ FILES : "locks"

    USERS {
        string id PK "UUID"
        string name "Full Name"
        string email UK "Unique Email"
        string password_hash "Bcrypt Hash"
        string role "admin | user"
        timestamp created_at "Registration Time"
    }

    FILES {
        string id PK "UUID"
        string owner_id FK "References USERS(id)"
        string original_filename "Display Filename"
        string stored_filename UK "Physical Disk Filename"
        string path "Server Storage Absolute Path"
        bigint file_size "Size in Bytes"
        string file_hash "SHA-256 Checksum"
        timestamp upload_time "Upload Date"
        integer version "Version Counter (1, 2, 3)"
        boolean is_locked "Lock Flag"
        string locked_by FK "References USERS(id)"
        timestamp locked_at "Lock Timestamp"
    }

    AUDIT_LOGS {
        string id PK "UUID"
        string user_id FK "References USERS(id)"
        string user_name "Snapshot User Name"
        string action "REGISTER, LOGIN, UPLOAD, etc."
        string filename "Target Filename"
        string details "Event Description"
        timestamp timestamp "Activity Timestamp"
    }
```

## Table Definitions

### 1. `users`
- `id`: Primary key (UUID string)
- `name`: Full display name
- `email`: Unique email identifier
- `password_hash`: Bcrypt hash
- `role`: Role enum (`admin`, `user`)
- `created_at`: Creation timestamp

### 2. `files`
- `id`: Primary key (UUID string)
- `owner_id`: Foreign key referencing `users(id)`
- `original_filename`: Original uploaded filename or versioned name (`resume_v2.pdf`)
- `stored_filename`: Unique storage filename generated by UUID
- `path`: Full path location on server filesystem
- `file_size`: Size in bytes
- `file_hash`: SHA-256 checksum used for duplicate detection
- `version`: Version integer counter
- `is_locked`: Boolean lock flag
- `locked_by`: User ID holding the lock

### 3. `audit_logs`
- `id`: Primary key (UUID string)
- `user_id`: Foreign key referencing `users(id)`
- `user_name`: Cached name of user performing action
- `action`: Audit action code (`REGISTER`, `LOGIN`, `LOGOUT`, `UPLOAD`, `DOWNLOAD`, `RENAME`, `DELETE`, `SEARCH`, `LOCK`)
- `timestamp`: Action timestamp
